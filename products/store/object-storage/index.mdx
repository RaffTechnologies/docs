<sub>Updated May 4, 2026</sub>

Object Storage is **S3-compatible** storage for unstructured data — media files, application backups, logs, static assets, machine-learning artifacts, anything you'd normally throw at an S3 bucket. Use any S3 SDK or tool against the Raff endpoint, organize objects into buckets, and serve them publicly or privately.

Today there is **one storage class — NVMe SSD** — tuned for high-performance access. A cost-effective HDD class is **coming**.

## Battle-tested — Raff runs on its own Object Storage

Object Storage isn't a new product we're selling and hoping holds up. It's the **same storage layer Raff itself runs on** in production. Every Raff service you interact with — the dashboard, the marketplace, image builds, audit logs, billing invoices — uses these buckets. The screenshot above shows real production buckets serving real Raff workloads.

That's the whole platform philosophy: **Raff runs on Raff**. Kubernetes, Virtual Machines, and Object Storage power the dashboard you're reading docs about right now. We validate every feature against our own load before exposing it to customers.

## S3 endpoint

Every bucket is reachable via the Raff S3-compatible endpoint:

```
https://<bucket-name>.s3.raffusercloud.com
```

Point any AWS-S3 SDK at that endpoint with your access keys and you're connected — no Raff-specific client library needed.
